# CHECKLIST OUTREACH DM (Curso IA, SaaS Marketing, IA Bulk)

## 1) Preparación (10-15 min por cuenta)
- [ ] Identificar [CLIENTE_IDEAL] y cargo (decisor/influencer)
- [ ] Encontrar [LOGRO] reciente verificable (post/noticia)
- [ ] Mapear industria, tamaño, stack (CRM/Ads/Analytics)
- [ ] Definir objetivo del DM (webinar, demo, trial, ejemplo)
- [ ] Elegir versión adecuada del DM (ver matriz en cada doc)
- [ ] Personalizar campos: [CLIENTE_IDEAL], [LOGRO], [INDUSTRIA], [FECHA], etc.
- [ ] Añadir prueba social relevante (caso/cliente similar)

## 2) Envío (Mejor timing)
- [ ] Martes-Jueves (10:00-12:00 o 14:00-16:00)
- [ ] 6-24h tras el [LOGRO] para capitalizar momentum
- [ ] Tono alineado al perfil (técnico vs negocio; formal vs casual)
- [ ] CTA único y claro (invitación, demo, prueba, ejemplo)

## 3) Seguimiento (si no responde)
- [ ] Seguimiento 1 (día 3-6): resumen de valor + CTA suave
- [ ] Seguimiento 2 (día 10-14): insight/caso específico
- [ ] Seguimiento 3 (día 20-30): último con recurso de valor
- [ ] Cerrar bucle si no hay respuesta (evitar overfitting)

## 4) Calidad y Cumplimiento
- [ ] Sin promesas absolutas; usar métricas con contexto
- [ ] No comparar negativamente con competidores
- [ ] Cuidar privacidad y fuentes de datos
- [ ] En industrias reguladas: mencionar compliance aplicable

## 5) Métricas a Registrar
- [ ] Tasa de respuesta por versión
- [ ] % conversiones a webinar/demo/trial
- [ ] Tiempo medio de respuesta
- [ ] Industria/versión más efectiva
- [ ] Comentarios cualitativos (objeciones comunes)

## 6) Plantillas rápidas de CTA
- Webinar: "¿Te reservo un lugar para el [FECHA]?"
- Demo: "¿Te va bien una demo de 15 min esta semana?"
- Trial: "¿Quieres activar una prueba de 14 días sin tarjeta?"
- Ejemplo: "¿Te muestro un ejemplo para [INDUSTRIA]?"

## 7) Errores comunes
- [ ] Enviar el mismo DM sin personalización
- [ ] Múltiples CTAs en un mensaje
- [ ] Follow-ups <72h o demasiado largos
- [ ] Falta de prueba social/métrica
