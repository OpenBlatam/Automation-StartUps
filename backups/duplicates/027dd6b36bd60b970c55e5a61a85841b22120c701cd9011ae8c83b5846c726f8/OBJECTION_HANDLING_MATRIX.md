# Matriz de Manejo de Objeciones (Objection Handling)

Estructura: Objeción → Respuesta breve → Prueba/Activo → Siguiente paso

| Objeción | Respuesta breve | Prueba/Activo | Siguiente paso |
|---|---|---|---|
| No es prioridad | Entiendo. Te comparto un caso de tu industria con impacto medible; si resuena, vemos 10-15 min. | Caso 1 página (industria) | Enviar caso + proponer 2 horarios |
| No hay presupuesto | Dos opciones: prueba gratuita con caso real o demo enfocada en ahorro de horas/costos. | Cálculo ROI rápido + testimonio | Activar trial o agendar demo |
| Ya usamos otra herramienta | Perfecto, solemos complementar integrándonos con [HERRAMIENTA] en [ÁREA]. Te muestro 2-3 diferencias clave. | Comparativa lado a lado | Demo con stack actual |
| Mándame info | Claro, ¿prefieres caso 1 página, video 3 min o ejemplo para tu industria? | Librería de recursos | Enviar recurso elegido + siguiente paso |
| No tenemos tiempo | Por eso priorizamos quick wins. Demo de 10-12 min con tu caso real para evaluar si vale seguir. | Agenda 2 slots + guion demo | Confirmar slot |
| Compliance/regulación | Cumplimos [NORMA]. Audit trail, control de acceso e integraciones seguras. Te enseño checklist. | LEGAL_PRIVACY_CHECKLIST + DPA | Enviar checklist + caso regulado |
| Datos/calidad | Integramos fuentes y generamos reporte con trazabilidad; validamos con tu muestra. | Ejemplo con datos ficticios | Enviar ejemplo + propuesta de POC |
| Resultados dudosos | Entiendo el escepticismo. Aquí tienes métricas agregadas y 2 casos comparables. | CASOS_EXITO_OUTREACH | Enviar 2 casos + 1 KPI esperado |
| Muy complejo implementar | Setup guiado en 1 hora. Integración con tu stack actual sin cambios drásticos. | AUTOMATION_PLAYBOOK + lista integraciones | Proponer sesión de setup |
| Necesitamos aprobación | Te envío un 1-pager para sponsor y un checklist para legal. | 1-pager + LEGAL_PRIVACY_CHECKLIST | Agendar con sponsor |

## Buenas prácticas
- Responde en <6h, ideal <2h
- Siempre ofrece 2 opciones de siguiente paso
- Adjunta un activo (caso, checklist, ejemplo)
- Documenta la objeción en CRM y resultado
